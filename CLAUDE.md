# HörenWeg — Инструкция для AI по созданию уроков аудирования

## О проекте

Статический сайт для изучения немецкого языка (A2) через транскрипции аудирования, интерактивные упражнения и AI-промты. Источник: экзамены Goethe Zertifikat A2 Hören. Работает offline через `file://` и на GitHub Pages.

**Структура:**
- `index.html` — оболочка сайта (CSS + JS + роутинг + все упражнения)
- `lessons/lesson-XX.js` — данные каждого урока (12 мини-текстов + словарь + промты)
- `tools/` — служебные Node.js скрипты для валидации (не публикуются в git)
- `tasks/` — сырые данные из расшаренных чатов Claude.ai (не публикуются в git)
- `Источники/` — методологические книги (не публикуются в git)
- `list/` — исходный список уроков в Excel (не публикуется в git)
- `.gitignore` — исключения для публикации на GitHub Pages

## КРИТИЧНО: Правила для sentenceTranslations

> **Это самый частый источник ошибок. Читай внимательно.**

Конструктор предложений разбивает `text` регулярным выражением:
```javascript
text.split(/(?<=[.!?])\s+/)
```

Затем ищет перевод по ТОЧНОМУ совпадению ключа в `sentenceTranslations`.

### Правило: ключ = ТОЧНО то, что выдаёт regex

**Как проверить:** Возьми поле `text` мини-текста и разбей его вручную по `.` `!` `?` с пробелом после. Каждый получившийся фрагмент должен быть ключом в `sentenceTranslations`.

**Типичные ошибки (из опыта 34 уроков):**

| Ошибка | Ключ в sentenceTranslations | Regex выдаёт из text |
|--------|----------------------------|---------------------|
| Обрезали начало | `"Ich war am Gardasee."` | `"Aber ich war am Gardasee."` |
| Разделили на два | `"Das Bein tut weh."` + `"Der Arm ist blau."` | `"Das Bein tut weh und der Arm ist blau."` |
| Пропустили предложение | *(нет ключа)* | `"Du kennst sie doch."` |
| **Сокращения** Dr./K./z.B. | `"Ich rufe von Dr. Koslowski an."` | `"Ich rufe von Dr."` + `"Koslowski an."` |
| **Даты** 31./20./65. | `"vor dem 65. Lebensjahr"` | `"vor dem 65."` + `"Lebensjahr in Rente."` |
| **Многоточие** ... | `"Weihnachten in Japan... ich weiß nicht."` | `"Weihnachten in Japan..."` + `"ich weiß nicht."` |

### ⚠️ Опасные паттерны в немецких текстах

Regex `/(?<=[.!?])\s+/` разбивает на КАЖДОЙ точке, включая:

1. **Сокращения:** `Dr.`, `K.`, `z.B.`, `usw.`, `ca.`
   - Пример: `"Hier spricht K. vom Reisebüro"` → regex даёт 2 фрагмента: `"Hier spricht K."` и `"vom Reisebüro"`
   - Решение: добавить ОБА фрагмента как отдельные ключи ИЛИ убрать точку в тексте

2. **Порядковые числительные с точкой:** `31.`, `20.`, `65.`, `12.`
   - Пример: `"bis zum 20. Februar"` → `"bis zum 20."` + `"Februar..."`
   - Решение: добавить оба фрагмента или заменить на текст (`"zwanzigsten"`)

3. **Многоточие `...`:** разбивает на `...` конце
   - Пример: `"In der Schule war... hat mein Lehrer..."` → 2 фрагмента
   - Решение: добавить оба фрагмента как отдельные ключи

4. **Кавычки в диалогах:** `?"` и `!"` НЕ вызывают split (`"` не в `[.!?]`)
   - Пример: `'Was meinst du?" Ich denke...'` остаётся ОДНИМ ключом
   - Это НЕ ошибка, просто помни: добавляй ключ целиком, не по кускам

### Автоматическая проверка

**Способ 1 — Node.js скрипт (быстро, все уроки сразу):**
```bash
node tools/find-missing.js
```
Скрипт `tools/find-missing.js` проверяет ВСЕ уроки и выводит список пропущенных ключей с номером текста.
Результат: `✅ Все sentenceTranslations на месте. 0 MISSING.` или список проблем.

**Способ 2 — В консоли браузера (проверка одного урока):**
```javascript
var lesson = LESSONS[0]; // или нужный урок
[1,2,3,4].forEach(function(teil) {
  var teilTexts = lesson.texts.filter(function(t){ return t.teil === teil; });
  var missing = [];
  teilTexts.forEach(function(t) {
    var trMap = {};
    for (var k in t.sentenceTranslations) trMap[k] = true;
    t.text.replace(/\n+/g, ' ').split(/(?<=[.!?])\s+/).forEach(function(s) {
      s = s.trim();
      if (s && !trMap[s]) missing.push('Text ' + t.id + ': ' + s);
    });
  });
  console.log('Teil ' + teil + ': ' + (missing.length ? missing.length + ' MISSING' : 'OK'));
  missing.forEach(function(m) { console.log('  ❌ ' + m); });
});
```

**Результат должен быть:**
```
Teil 1: OK
Teil 2: OK
Teil 3: OK
Teil 4: OK
```

## Интерактивные упражнения (на сайте, без AI)

Все упражнения разбиты по 4 Teil. Данные берутся из `vocabulary[]` и `text` каждого мини-текста. Код в `index.html`.

### 4 типа упражнений × 4 Teil = 16 аккордеонов:

#### 🎯 Викторина по словам (до 24 слов на Teil)
- DE→RU / RU→DE, 4 варианта ответа, прогресс-бар
- Слова: из `vocabulary[]` текстов данного Teil (функция `getVocabByTeil`)
- Дистракторы: из ПОЛНОГО словаря урока (все Teil) — для разнообразия
- Подсказка: поле `example` — предложение из транскрипции
- Максимум 24 слова, но может быть меньше

#### 🧩 Сопоставление слов (до 10 пар на Teil)
- 2 колонки: DE (голубой) и RU (жёлтый), клик-соединение
- Слова: из `getVocabByTeil(lesson, teil).slice(0, 10)`
- Прогресс-бар

#### 🎨 Артикли и род (до 12 существительных на Teil)
- Существительное без артикля + 3 кнопки der/die/das
- Только слова с `gender` из текстов данного Teil
- **ВАЖНО:** поле `de` у существительных ДОЛЖНО начинаться с `der`/`die`/`das`
- Прогресс-бар, цвета: der=синий, die=красный, das=зелёный

#### 🧱 Конструктор предложений (до 4 предложений на Teil, макс 10-12 слов)
- Собери предложение из перемешанных слов
- Предложения: из `text` текстов данного Teil (4-12 слов)
- Подсказка: русский перевод из `sentenceTranslations` (поле `literary`)
- Кнопки: Подсказка, Убрать, Проверить, Сброс, Ответ
- **КРИТИЧНО:** каждое предложение ДОЛЖНО иметь ключ в `sentenceTranslations`

## Правила для данных

### vocabulary (для каждого текста):
- Уровень **A2-B1** — ключевые слова для понимания текста
- НЕ включать базовые A1: ich, du, sein, haben, gehen, kommen, ja, nein
- Роль преподавателя: выбирай слова, без которых ученик не поймёт текст
- **Существительные:** ВСЕГДА `der/die/das` + `gender` + `plural` + `example`
- **Глаголы:** инфинитив, `type: 'verb'`, `forms`, `example`, БЕЗ gender
- **Прилагательные:** `type: 'adj'`, `example`, БЕЗ gender
- **example:** ОБЯЗАТЕЛЬНО — короткое предложение ИЗ ТРАНСКРИПЦИИ, где встречается слово

### sentenceTranslations (для каждого текста):
- **ВСЕ предложения** из поля `text` должны иметь перевод
- Ключ = ТОЧНАЯ строка, которую выдаёт `text.split(/(?<=[.!?])\s+/)`
- Значение = объект `{ literal: '...', literary: '...' }`
- `literal` = дословный (пословный) перевод
- `literary` = литературный (смысловой) перевод
- **Проверяй regex-ом после создания!** (см. автоматическую проверку выше)

### question (для каждого текста):
- 1 вопрос на немецком + перевод на русский
- 3 варианта ответа (A/B/C) на немецком + русский
- `correct` = 0-based индекс правильного
- `explanation` = цитата из текста, доказывающая ответ

- Каждый таб (Словарь, Диалог, Сократ, Синонимы, Лефевр) показывает **4 аккордеона** — по одному на Teil

- Грамматика (`grammar`) — один промт на весь урок (не разбит по Teil)

> **СНАЧАЛА ПОКАЖИ КАК — ПОТОМ СПРАШИВАЙ.**

## Как писать промты

> Каждый промт (кроме grammar) создаётся отдельно для каждого Teil.
> Промт должен упоминать конкретные ситуации и слова ТОЛЬКО из своего Teil.
> Переменные `{lessonText}` и `{vocabularyList}` подставятся автоматически.

### 📝 Перевод + грамматика (`grammar`) — один на весь урок
- Ученик переводит транскрипцию по предложениям
- Образец → ученик сам → мини-уроки грамматики каждые 3-4 предложения
- Правильный ответ = макс 2 строки. Ошибка = макс 4 строки.
- Темп > идеальность

### 📚 Словарь (`vocabulary_t1`...`vocabulary_t4`) — по Teil
- 5 этапов: знакомство → викторина → перевод → контекст → мини-пересказ
- Артикли обязательны: "Kollege" без артикля = неправильно
- Примеры и контекст — из текстов данного Teil
- Каждый промт ~1400-1600 символов

### 🎭 Диалог (`dialogue_t1`...`dialogue_t4`) — по Teil
- 4 фазы: повторение → викторина → достройка → свободный диалог
- Цель: РАЗГОВОРИТЬ, не проверить
- Фразы для повторения и ситуации — из текстов данного Teil
- Каждый промт ~1300-1600 символов

### 🏛 Сократ (`socratic_t1`...`socratic_t4`) — по Teil
- Ученик САМ открывает грамматику через вопросы
- Эленхус по Farnsworth: при ошибке не исправляй — задай вопрос-противоречие
- 1-2 предложения из данного Teil с цепочками вопросов
- ВСЁ на русском. Немецкий — только примеры.
- Методология: `Источники/The_Socratic_Method.txt`
- Каждый промт ~1500-1600 символов

### 🔄 Синонимы (`synonyms_t1`...`synonyms_t4`) — по Teil
- Викторина A/B/C (2-4 вопроса на Teil)
- Ученик ВЫБИРАЕТ, а не придумывает
- Трансформации: синоним глагола, Aktiv→Passiv, два предложения→одно с weil
- Каждый промт ~1800-2200 символов

### 🪞 Лефевр (`lefebvre_t1`...`lefebvre_t4`) — по Teil
- Пересказ через "внутреннюю карту": КТО? ЧТО? ГДЕ? КОГДА?
- Ступени: образец → вместе → сам → рефлексия (адаптированы под кол-во текстов Teil)
- Методология рефлексии: `Источники/Конфликтующие структуры.txt`
- Каждый промт ~1500-1700 символов

## Общие правила для ВСЕХ промтов

1. **Язык общения — русский.** Немецкий только в примерах.
2. **Сначала покажи, потом спрашивай.** Образец → вместе → сам.
3. **Краткость.** Правильный = 2 строки. Ошибка = 4 строки.
4. **Темп > идеальность.** Лучше пройти весь текст с ошибками, чем застрять.
5. **Чувствуй ученика.** Уверенный → ускоряй. Слабый → помогай.
6. **Хвали коротко.** "Genau!" / "Super!" — без длинных похвал.
7. **Максимум 1 исправление за ответ.**
8. **Юмор приветствуется.**
9. **Артикли обязательны** для существительных — всегда.

### Batch-процесс (для массового добавления уроков через агентов)

Когда запускаешь несколько агентов параллельно для создания уроков:

1. **Скачивание контента** (Chrome MCP, последовательно):
   - `navigate → wait 6-8 сек → javascript_tool: download blob` → файл сохраняется в `Downloads/lesson-XX-raw.txt`
   - Chrome MCP работает только с одной вкладкой, параллелить нельзя
   - После скачивания: `cp Downloads/lesson-XX-raw.txt tasks/lesson-XX-raw.txt`

2. **Создание данных** (параллельные агенты, по 1 на урок):
   - Каждый агент получает: сырой файл + lesson-01.js (эталон) + CLAUDE.md
   - **ВАЖНО:** инструктируй агента СКОПИРОВАТЬ геттеры из lesson-01.js verbatim
   - Agents без Bash не могут валидировать `node -c` — делай это сам после

3. **Валидация после агентов** (через Bash):
   - `node -c lessons/lesson-XX.js` для каждого файла
   - Запусти сервер + JS-тест в браузере: `0 MISSING sentenceTranslations`
   - Типичные проблемы: отсутствие геттеров, Dr./даты/многоточия в regex

4. **Массовое исправление**:
   - Если много уроков имеют одинаковую проблему (напр. пропущенные геттеры) — 1 агент фиксит все
   - Для ключей sentenceTranslations: используй JS-анализатор в браузере чтобы найти ТОЧНЫЕ missing фрагменты
   - Передавай агенту список конкретных ключей с готовыми переводами
